The chart below outlines the compatibility between different versions of Elastic APM agents and extensions with the APM integration.

| Language | ((apm-agent)) version | APM integration version |
|---|---|---|
| **APM AWS Lambda extension** | `1.x` | ≥ `8.2` |
| **Go agent** | `1.x` | ≥ `6.5` |
| |`2.x` | ≥ `6.5` |
| **iOS agent** | `0.x` | ≥ `7.14` |
| **Java agent**((empty))^1^ | `1.x | ≥ `6.5` |
| **.NET agent** | `1.x` | ≥ `6.5` |
| **Node.js agent** | `3.x` | ≥ `6.6` |
| **PHP agent** | `1.x` | ≥ `7.0` |
| **Python agent** | `6.x` | ≥ `6.6` |
| **Ruby agent** | `3.x` | ≥ `6.5` |
| |`4.x` | ≥ `6.5` |
| **JavaScript RUM agent** | `4.x` | ≥ `6.5` |
| |`5.x` | ≥ `7.0` |

1: Java agent \< 1.43.0 not fully compatible with APM Server >= 8.11.0. Read more in <DocLink slug="/en/observability/apm-known-issues" section="apm-empty-metricset-values">Known issues</DocLink>.
