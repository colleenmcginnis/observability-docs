## APM version 8.6.0

[View commits](https://github.com/elastic/apm-server/compare/v8.5.3\...v8.6.0)

<div id="breaking_changes"></div>

### Breaking Changes
- `apm-server.decoder.*` stack monitoring metrics are no longer recorded [9210]({pull}9210)
- `ecs.version` is no longer added to document `_source`; it is added as a `constant_keyword` field [9208]({pull}9208)
- `context.http.response.*_size` fields now enforce integer values [9429]({pull}9429)
- `observer.id` and `observer.ephemeral_id` are no longer added to APM documents [9412]({pull}9412)
- `timeseries.instance` has been removed from transaction metrics docs; it was never used [9565]({pull}9565)

<div id="deprecations"></div>

### Deprecations
- `apm-server apikey` commands have been deprecated. API Keys should be managed through Kibana or the Elasticsearch REST API [9446]({pull}9446)

<div id="bug_fixes"></div>
### Bug fixes
- Change ownership of apm-server.yml and data/ to apm-server user in Docker image [9604]({pull}9604)

<div id="intake_api_changes"></div>

### Intake API Changes
- <DocBadge template="technical preview" />  Extend logs v2 intake API to support ECS logging fields [9349]({pull}9349)
- <DocBadge template="technical preview" />  Add support for string timestamp format(`2006-01-02T15:04:05.999-0700`) [9376]({pull}9376)
- `context.http.response.*_size` fields have been changed from floating-point to integer types [9429]({pull}9429)

<div id="added"></div>

### Added
- Service metrics now support global labels [9248]({pull}9248)
- OpenTelemetry GRPC Spans from the Javascript API/SDK/Instrumentations are now correctly transformed into transactions with type=`request` [9308]({pull}9308)
- Improve Elasticsearch output performance, particularly when compression is enabled (default) [9318]({pull}9318)
- Java attacher support for macOS [9413]({pull}9413)
- Improve Elasticsearch output performance in instances with more than 6 cores [9393]({pull}9393)
- Tune internal server configuration based on available cgroup or machine RAM, improving resource utilization and performance [9358]({pull}9358)
- Disallow auto-scaling of active indexers when Elasticsearch 429 response rate exceeds 1% of total requests issued [9463]({pull}9463)
- We now record `transaction.representative_count` and `span.representative_count` -- the inverse sample rate [9458]({pull}9458)
