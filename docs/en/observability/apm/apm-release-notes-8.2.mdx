## APM version 8.2.0

[View commits](https://github.com/elastic/apm-server/compare/v8.1.3\...v8.2.0)

<div id="breaking_changes"></div>

### Breaking Changes
- APM Server now emits events with `event.duration` and renames the field to `<event>.duration.us` in an ingest pipeline [7261]({pull}7261)
- Removed `observer.version_major` [7399]({pull}7399)
- APM Server no longer ships with System V init scripts or the go-daemon wrapper; use systemd [7576]({pull}7576)

<div id="deprecations"></div>

### Deprecations
- APM Server support for 32-bit architectures has been deprecated and will be removed in a future release [7678]({pull}7678)

<div id="bug_fixes"></div>
### Bug fixes
- Do not overwrite `service version` if no transaction/error/... specific `service.name` is given [7281]({pull}7281)
- Set transaction name to `"<method> unknown route"` for unknown HTTP routes in APM Server instrumentation [7679]({pull}7679)

<div id="intake_api_changes"></div>

### Intake API Changes
- Support for `faas.name` and `faas.version` added to intake and transaction metrics [7427]({pull}7427)
- If proxy headers are present: `source.ip` and `client.ip` set to header values, and `source.nat.ip` set to incoming request IP [7444]({pull}7444)
- Updated intake v2 with support for `links` in `transaction` and `span` events [7553]({pull}7553)
- If an agent sends a transaction or span with type unspecified, the server now sets a default value of "unknown" [7688]({pull}7688)
- Content-Type and Content-Encoding are no longer required for intake [7686]({pull}7686)

<div id="added"></div>

### Added
- Implemented translation of OpenTelemetry host system metrics (CPU utilization / Memory usage) [7090]({pull}7090)
- Added support for storing OpenTelemetry span links as `span.links` [7291]({pull}7291)
- seccomp is disabled by default [7308]({pull}7308)
- Added data stream namespace configuration for standalone with `apm-server.data_streams.namespace` [7314]({pull}7314)
- `apm-server` artifacts now have the apm java-attacher.jar packaged alongside them [6593]({pull}6593)
- Run the java attacher jar when configured and not in a cloud environment [6617]({pull}6617)
