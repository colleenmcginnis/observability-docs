TLS is disabled by default.
When TLS is enabled for APM Server inbound communication, agents will verify the identity
of the APM Server by authenticating its certificate.

When TLS is enabled, a certificate and corresponding private key are required.
The certificate and private key can either be issued by a trusted certificate authority (CA)
or be <DocLink slug="/en/observability/apm-agent-tls" section="use-a-self-signed-certificate">self-signed</DocLink>.

## Use a self-signed certificate

<div id="apm-agent-self-sign-1"></div>

### Step 1: Create a self-signed certificate

The ((es)) distribution offers the `certutil` tool for the creation of self-signed certificates:

1. Create a CA: `./bin/elasticsearch-certutil ca --pem`. You'll be prompted to enter the desired
    location of the output zip archive containing the certificate and the private key.

1. Extract the contents of the CA archive.
1. Create the self-signed certificate: `./bin/elasticsearch-certutil cert --ca-cert <path-to-ca-crt>/ca.crt --ca-key <path-to-ca-key>/ca.key --pem --name localhost`
1. Extract the certificate and key from the resulted zip archive.

### Step 3: Configure APM agents

When the APM server uses a certificate that is not chained to a publicly-trusted certificate
(e.g. self-signed), additional configuration is required in the ((apm-agent)):

* **Go agent**: certificate pinning through [`ELASTIC_APM_SERVER_CERT`](((apm-go-ref))/configuration.html#config-server-cert)
* **Python agent**: certificate pinning through [`server_cert`](((apm-py-ref))/configuration.html#config-server-cert)
* **Ruby agent**: certificate pinning through [`server_ca_cert`](((apm-ruby-ref))/configuration.html#config-ssl-ca-cert)
* **.NET agent**: [`ServerCert`](((apm-dotnet-ref))/config-reporter.html#config-server-cert)
* **Node.js agent**: custom CA setting through [`serverCaCertFile`](((apm-node-ref))/configuration.html#server-ca-cert-file)
* **Java agent**: adding the certificate to the JVM `trustStore`.
    See [APM Server authentication](((apm-java-ref))/ssl-configuration.html#ssl-server-authentication) for more details.

We do not recommend disabling ((apm-agent)) verification of the server's certificate, but it is possible:

* **Go agent**: [`ELASTIC_APM_VERIFY_SERVER_CERT`](((apm-go-ref))/configuration.html#config-verify-server-cert)
* **.NET agent**: [`VerifyServerCert`](((apm-dotnet-ref))/config-reporter.html#config-verify-server-cert)
* **Java agent**: [`verify_server_cert`](((apm-java-ref))/config-reporter.html#config-verify-server-cert)
* **PHP agent**: [`verify_server_cert`](((apm-php-ref-v))/configuration-reference.html#config-verify-server-cert)
* **Python agent**: [`verify_server_cert`](((apm-py-ref))/configuration.html#config-verify-server-cert)
* **Ruby agent**: [`verify_server_cert`](((apm-ruby-ref))/configuration.html#config-verify-server-cert)
* **Node.js agent**: [`verifyServerCert`](((apm-node-ref))/configuration.html#validate-server-cert)
