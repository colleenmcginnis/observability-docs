## APM version 8.3.0

[View commits](https://github.com/elastic/apm-server/compare/v8.2.2\...v8.3.0)

<div id="breaking_changes"></div>

### Breaking Changes
- [experimental] Removed support for experimental firehose endpoint for CloudWatch metric streams [8109]({pull}8109)

<div id="deprecations"></div>

### Deprecations
- `span.context.destination.service.resource` is deprecated [7925]({pull}7925)

<div id="bug_fixes"></div>
### Bug fixes
- Propagate datastream namespace changes from apm-integration into server [8176]({pull}8176)
- Correct the `libbeat.output.write.bytes` metric to count the written bytes after the compressed writer has been flushed [8221]({pull}8221)
- Allow 100% tail-based sampling [8233]({pull}8233)

<div id="intake_api_changes"></div>

### Intake API Changes
- For OpenTelemetry exception span events, stack traces that cannot be parsed will now be stored in `event.stack_trace` [7706]({pull}7706)
- Support for ingesting `service.target.type` and `service.target.name` added to intake API for spans [7870]({pull}7870)
- Derive `service.target.{type, name}` fields for older agents from `span.context.destination.service.resource` [7925]({pull}7925)
- Support `service.target.*` in metrics aggregation [7924]({pull}7924)

<div id="added"></div>

### Added
- System, process, and well-defined runtime metrics are now sent to the shared `metrics-apm.internal-<namespace>` data stream [7882]({pull}7882)
- Number of parallel bulk requests are now configurable via `output.elasticsearch.max_requests` [8055]({pull}8055)
- OTLP/HTTP protocol is now supported [8156]({pull}8156)

<div id="performance_improvements"></div>

### Performance improvements
- Limit the number of gomaxprocs when CPU limits have been set within a cgroup, yielding significant performance improvements, particularly when the APM Server experiences high traffic from a high number of APM agents [8175]({pull}8175) [8278]({pull}8278)
