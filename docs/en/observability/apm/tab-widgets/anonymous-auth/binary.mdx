
When an <DocLink slug="/en/observability/apm-api-key">API key</DocLink> or <DocLink slug="/en/observability/apm-secret-token">secret token</DocLink> is configured,
anonymous authentication must be enabled to collect RUM data.
To enable anonymous access, set either <DocLink slug="/en/observability/apm-configuration-rum" section="enable-rum">`apm-server.rum.enabled`</DocLink> or
<DocLink slug="/en/observability/apm-configuration-anonymous" section="anonymous-agent-access">`apm-server.auth.anonymous.enabled`</DocLink> to `true`.

Because anyone can send anonymous events to the APM Server,
additional configuration variables are available to rate limit the number anonymous events the APM Server processes;
throughput is equal to the `rate_limit.ip_limit` times the `rate_limit.event_limit`.

See <DocLink slug="/en/observability/apm-configuration-anonymous">Configure anonymous authentication</DocLink> for a complete list of options and a sample configuration file.
