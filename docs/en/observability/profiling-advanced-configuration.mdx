<div id="profiling-advanced-configuration"></div>

After completing the steps in the <DocLink id="enObservabilityProfilingGetStarted">Get started</DocLink> documentation, you may need to continue to more advanced configuration for your deployment.
See the following sections for more information:

* <DocLink id="enObservabilityProfilingTagDataQuery">Tag data for querying</DocLink>: Tag data collected by the Universal Profiling Agent into multiple logical groups so they can be queried in Kibana.
* <DocLink id="enObservabilityProfilingAddSymbols">Add symbols for native frames</DocLink>: Push symbols to your cluster so you can see function names and line numbers in traces of applications written in programming languages that compile to native code (C, C++, Rust, Go, etc.).
* <DocLink id="enObservabilityProfilingUseAProxy">Use a proxy</DocLink>:  Set up an HTTP proxy if your infrastructure Universal Profiling Agent installation needs one to reach ((ecloud)).
* <DocLink id="enObservabilityProfilingNoKernelVersionCheck">Override kernel version check </DocLink>: Configure the Universal Profiling Agent to bypass the kernel version compatibility check.
